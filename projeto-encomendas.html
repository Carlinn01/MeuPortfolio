Sistema de Gerenciamento de Encomendas

Sobre o Projeto

Este sistema foi desenvolvido para a loja Tio Léo Utilidades com o objetivo de gerenciar de forma eficiente todas as encomendas realizadas pelos clientes. O sistema permite cadastrar, visualizar, atualizar e controlar o status de cada encomenda, facilitando o acompanhamento e organização do negócio.

O projeto foi criado para resolver problemas de organização e controle de pedidos, permitindo que a loja tenha um melhor controle sobre as encomendas em andamento e concluídas, além de facilitar a comunicação com os clientes sobre o status de seus pedidos.

Objetivo

O objetivo principal deste sistema é fornecer uma solução completa e intuitiva para o gerenciamento de encomendas da loja, permitindo:

  • Cadastro e controle de encomendas
  • Gerenciamento de status (pendente, em produção, concluído, entregue)
  • Organização e busca de informações
  • Interface amigável e fácil de usar
  • Armazenamento seguro dos dados

O Que Foi Utilizado

Java: Utilizei Java como linguagem principal para desenvolvimento da lógica de negócio e funcionalidades do sistema. O Java proporciona uma base sólida e confiável para aplicações desktop.

SQLite: Escolhi SQLite como banco de dados por ser leve, eficiente e não requerer configuração de servidor. É ideal para aplicações desktop, armazenando todos os dados de forma organizada e segura.

Swing: Para a interface gráfica, utilizei o Swing, framework nativo do Java que permite criar interfaces desktop modernas e responsivas. A interface foi projetada para ser intuitiva e fácil de usar.
